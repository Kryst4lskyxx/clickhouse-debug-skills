## Problem & goal

`clickhouse-debug` (v0.5.0) is a mature diagnosis skill: Prometheus triage â†’
read-only `system.*` forensics â†’ source-confirmed root cause, every probe
resource-capped via `chq.sh`. Three improvements were requested, spanning three
buckets:

1. **Less in-use friction** â€” collapse the manual "gather inputs / confirm setup
   / version-match" prose into an executed check.
2. **Safety enforcement** â€” make the "route through `chq.sh`, never raw curl"
   rule *enforced*, not just documented.
3. **Companion coverage** â€” auto-detect which companion suites are installed, and
   make the symptomâ†’specialist routing executable rather than recited.

Two sub-mechanisms were explicitly chosen for the companion piece: **availability
detection** and **smarter routing**. Worked integration examples and new incident
signatures/eval scenarios were explicitly deferred.

## Key decisions

- **Layered distribution.** The skill ships two ways: `npx skills add` (copies
  only `skills/clickhouse-debug/`, cross-agent) and the Claude Code plugin
  (`.claude-plugin/`). Hooks execute only for plugin installs in Claude Code.
  Decision: a **portable core** (scripts that work everywhere) **plus** a Claude
  Code **plugin hook** as an enforcement bonus layer on top.
- **Availability detection is model-side.** The agent already has its installed
  skill list in context; a bash scan of `~/.claude/skills` is unreliable across
  agents and install layouts. Scripts do only what bash does reliably
  (source-tree version, endpoint reachability, version match, topology); the
  agent inspects its own skill list for companion availability.
- **No new query logic.** `preflight.sh` calls `chq.sh` so the `agent-query-safety`
  caps are inherited, not duplicated.
- **Routing source of truth is a TSV, guarded by a test** (not generated into the
  skill body â€” too fragile). The human-readable SKILL.md table stays; a test
  asserts it can't drift from the TSV.

## Component detail

### `preflight.sh`

Pure bash, `set -euo pipefail`, sourced from `.chenv` like the other scripts.
Run as step 0: `source ./.chenv && ./preflight.sh`. Reports, in order:

1. **Source tree** â€” find `cmake/autogenerated_versions.txt` (or `git describe`)
   in CWD, extract source version. Absent â†’ `WARN: not in a CH source tree;
   source-confirmation step unavailable.`
2. **Endpoints** â€” `curl -sk $CH_URL/ping` (expects `Ok.`) + a Prometheus health
   probe; report reachable/not, retry-once on transient curl failure (same ethos
   as `chq.sh`).
3. **Version match (automated)** â€” run `SELECT version()` *through `chq.sh`*, diff
   against source version â†’ `PASS` / `WARN: source X vs live Y, line numbers may
   differ`.
4. **Topology** â€” list clusters from `system.clusters`; if one obvious non-default
   cluster, print node count via `clusterAllReplicas(<cluster>, system.one)` so
   fan-out cap-sizing is known up front.
5. **Summary** â€” a single `STATUS: READY` / `STATUS: BLOCKED: <reasons>` line the
   agent can act on, plus a one-line reminder to do the model-side companion
   availability check.

**Behavior contract:**
- Exit `0` even when not fully ready (emit a `STATUS:` line rather than failing
  the agent's flow). Only a missing `CH_URL` is fatal (non-zero), matching
  `chq.sh`'s own `CH_URL` requirement.
- Honor eval replay mode (`CH_REPLAY_DIR`) via the shared `_fixture.sh` hook, so
  preflight is fixture-testable offline like the other probes.

### `routing.tsv` + `route.sh`

`references/routing.tsv` â€” one row per signature, tab-separated:

```
pattern	primary_reference	specialist	note
```

`pattern` is an error code (e.g. `CANNOT_SCHEDULE_TASK`) or a symptom keyword
(e.g. `cache`, `dictionary`, `kafka`, `mutation`, `grant`). Rows mirror the
existing SKILL.md symptomâ†’specialist table.

`scripts/route.sh <term>` â€” case-insensitive lookup; prints matching rows
(primary reference file + specialist to invoke + note). No match â†’ suggests
`altinity-expert-clickhouse-overview` (health-snapshot entry point). The agent can
run it on any pasted error code instead of guessing.

`scripts/tests/routing.test.sh` â€” drift guard: every specialist named in
`routing.tsv` also appears in SKILL.md's routing table and vice-versa.

### Curl-guard hook

`scripts/hooks/pretooluse-chq-guard.sh` â€” `PreToolUse` hook on the `Bash` tool.
**Blocks (exit 2) only a clear violation:** a `curl` to a CH port
(`8123`/`8443`/`9000`/`9440`) that **carries a query** (`query=` / `--data`)
**and is not `chq.sh`**. stderr explains: route through `chq.sh` so the
`agent-query-safety` caps apply.

**Fail-open by design:** bare `curl $CH_URL/ping` (no query â†’ allowed, preflight
relies on it); non-CH bash untouched; ambiguous cases pass. Thin grep; only the
precise dangerous case is blocked.

**Layering:** the hook *script* lives inside the skill dir (`scripts/hooks/`) so
it travels via npx and those users can wire it manually (documented). The
*wiring* ships in the plugin only â†’ auto-active for plugin installs, absent for
npx.

## Risks / open questions
